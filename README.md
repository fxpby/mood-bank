# 情感储蓄罐

情感储蓄罐是一个移动端优先的本地 PWA，用来持续看见、整理和积累亲密关系与自我关系中的重要线索。它既可以在被触发、想检查、想解释、睡前复盘时帮助你先稳住自己，也可以在温暖互动、被理解、想学习怎么爱与被爱、想练习边界和表达时，帮你把健康的看见存下来，逐渐积累自我稳定和亲密关系能力。它不是关系评分器，也不是心理咨询或危机干预工具。

当前版本已经从 P0 扩展到 P1/P2 轻量功能：除了「我被触发了」「回到自己」「记录互动」，还包含信号检查、草稿自检、复杂来信拆解、稍后发现点、小练习、账户明细，以及多个轻量心理觉察分支。整体目标不是只处理危机，而是帮助用户在日常波动中反复练习：识别情绪、校准情绪含义、区分事实和解释、看见旧模式、保存温暖证据、练习主体性、学习健康关系。

## 这个应用适合用来做什么

- **看见自己**：记录事实、情绪、身体反应、解释、冲动和下一步，而不是只停留在混乱感里。
- **积累稳定**：把回到自己的动作、锚点、小练习和有效的低压选择存下来，让稳定不是一次性技巧，而是可以反复回看的经验。
- **校准情绪**：把恐惧、羞耻、愤怒、脆弱等情绪看作信号和保护，而不是敌人；看见它保护什么，也看见它可能推你做什么过快反应。
- **保存健康连接证据**：把被看见、被理解、互相照顾、真实修复、尊重边界等可观察线索存下来，避免只在焦虑时回看关系。
- **练习表达和边界**：在发消息前看动机、承受度、姿态和边界，不追求操控对方回应，而是练习更清楚、更低压地表达自己。
- **学习亲密关系能力**：在吸引、幻灭、自省、修复、磨合和普通照顾里，慢慢学习爱与被爱的方式，而不是只追逐新鲜感或确定感。
- **把发现点放进稍后**：一次事件里看见很多线索时，先存下来，不要求当场分析完。
- **建立个人行动菜单**：用很小的练习把觉察落到行动，不把照顾自己变成任务或打卡压力。

## 使用前先知道

- 所有数据只保存在当前浏览器的 `localStorage` 中。
- 没有后端、登录、云同步、遥测、推送或 AI 分析。
- 清理浏览器数据、更换设备、使用隐私模式或浏览器权限变化，都可能导致记录丢失。
- 导出 / 导入入口目前只是占位，暂未开放。
- 应用不会读取聊天软件、社交媒体、在线状态、已读回执、回复速度或外部 App 数据。
- 应用不会判断对方真实想法，不提供关系结论、真爱测试、兼容性评分、去留建议或创伤诊断。

## 第一次使用

首次打开会进入 `/setup`。

你需要完成两件事：

1. 设置一个「情感空间」
   - 可以是「某段关系」
   - 也可以是「我和自己」
   - 名称只保存在本机，用来帮助你区分当前观察对象

2. 确认本地保存边界
   - 记录只存在当前浏览器
   - 应用不是心理治疗、医疗建议或危机干预
   - 如果担心自己或他人的即时安全，应优先联系可信任的人、当地紧急服务或专业支持

完成后会进入首页 `/home`。

## 首页怎么用

首页是日常入口，也是高波动时的快速入口。第一屏先放最常用的稳定和记录动作，让你既能在被触发时少加重一点，也能在平稳或温暖时把有价值的看见存下来。

首页主要区域：

- 当前空间：显示当前观察的是哪段关系或哪部分自己。
- 今日状态：显示一个简短的状态说明。
- 四个主入口：
  - 我被触发了
  - 想检查信号
  - 草稿自检
  - 回到自己
- 记录互动：进入记录列表或新建记录。
- 校准一个情绪：把情绪当作信使，不急着让它开车。
- 三个储蓄罐：连接、自己、能量。
- 最近存下：最近一条记录。
- 今天的锚点：可以带着一句话进入「回到自己」。

底部导航：

- 首页：回到当前状态和主入口。
- 记录：查看或新建互动记录。
- 稍后：查看发现点、问题、话题和行动想法。
- 练习：查看或创建小练习。
- 设置：管理空间、本地保存和删除数据。

## 三个储蓄罐

三个储蓄罐不是关系总分，也不是对你或对方的评价。它们只从你保存下来的记录和练习里派生，用来帮助你回看「发生了哪些可观察的变化」。

### 连接

连接罐只记录比较具体的连接证据：

- 对方有具体回应、理解或在场
- 一段互动里有可观察的互相靠近
- 如果当前空间是「我和自己」，也可以记录和自己的真实接触

不会进入连接罐的内容：

- 猜测、期待、幻想
- 沉默本身
- 对方是否在线、是否已读、回复快慢
- 单纯的「我觉得关系完了」或「我觉得对方一定在乎」

### 自己

自己罐记录主体性回来的时刻：

- 区分事实和解释
- 没有让冲动直接开车
- 选择一个自己能完成的小动作
- 完成或部分完成小练习
- 看见旧感觉、内部审判者或边界压力

它不是自尊分数，也不评价你够不够好。

### 能量

能量罐记录消耗或恢复：

- 回到自己后轻了一点
- 某次记录显示更累、更重
- 完成小练习后有一点恢复

能量低不是失败，只是提醒你现在可能需要更轻的动作。

## 常见场景使用手册

### 场景 1：我被触发了

入口：首页 -> `我被触发了`

适合这些时刻：

- 没有回复后很慌
- 想补发解释
- 想确认关系有没有变冷
- 看到某个信号后开始失控
- 收到长消息后身体很紧
- 争执后想立刻做点什么

流程：

1. 先抓住事实
   - 只写摄像头能拍到的部分
   - 不读心、不推断未来

2. 看见身体和情绪
   - 选择身体反应、情绪和强度
   - 可以选「说不清」

3. 看见冲动
   - 比如马上回复、检查信号、反复重读、补发解释、指责、撤退、拯救、失眠复盘

4. 选一个由我能完成的下一步
   - 延迟 10 分钟
   - 保存草稿不发
   - 记录事实
   - 把话题放进稍后
   - 做五感落地
   - 回到自己

完成后可以：

- 直接完成
- 高激活时进入「回到自己」
- 保存为快速记录
- 如果选择了「把话题放进稍后」，可以存入稍后

### 场景 2：我想检查信号

入口：首页 -> `想检查信号`

适合这些时刻：

- 想刷新消息、动态或状态
- 想通过一个信号确认对方在不在乎
- 想知道连接有没有变冷
- 想补发一句来换确定感

流程：

1. 我想通过检查确认什么？
2. 如果看到好信号，我接下来可能会怎样？
3. 如果没有看到想要的信号，我可能会怎样？
4. 先给自己 10 分钟，不把决定交给信号

你可以选择：

- 五感落地
- 喝水 / 洗手
- 把手机扣下 10 分钟
- 保存草稿到明天
- 写下事实，不写结论
- 看一个已存下的温暖证据
- 回到自己
- 我还是想检查

「我还是想检查」不是失败。应用不会惩罚你，也不会记录 streak 或禁止你检查。它只帮助你看见：我在期待什么，检查后发生了什么。

可保存内容：

- 一个发现点，进入「稍后」
- 不会因为保存发现点改变三个储蓄罐

### 场景 3：我写了一段话，不知道该不该发

入口：首页 -> `草稿自检`

适合这些时刻：

- 写了很长一段解释
- 想道歉、修复、表达请求或说边界
- 担心对方不回应
- 害怕发出去后一直盯回复
- 感觉自己在证明、拯救或控诉

流程：

1. 草稿输入
2. 检查当前状态
3. 看写这段话的动机
4. 看如果暂时没有回应能否承受
5. 看内容风险
6. 看姿态：创造者 / 引导者 / 挑战者，或受害者 / 拯救者 / 迫害者
7. 预演发出去以后可能发生什么
8. 得到一个确定性建议

可能建议：

- 可以发：足够真实，也足够低压
- 减轻一点：表达有价值，但压力偏高
- 先存草稿：现在回应期待太高
- 先私下记录：这段可能更像写给自己看的
- 改成边界表达：核心是需要或限度
- 先回到自己：现在不适合做发送决定

这个流程不会：

- 帮你重写成更容易获得回应的话
- 预测对方会怎么想
- 生成最佳回复
- 替你发送

可保存内容：

- 本地草稿
- 发现点
- 私下记录
- 稍后话题

也可以从这里进入：

- 回到自己
- 边界清晰
- 旧感觉 / 内部审判者
- 自我关怀
- 主体性转换
- 看见证据
- 健康关系学习
- 连接恒常

### 场景 4：我收到很多内容，不知道怎么接

入口：记录 -> `收到一段很长/很暖/信息很多的话`

适合这些时刻：

- 收到很暖、很长、信息很多的内容
- 感觉被看见，但也有压力
- 想认真回应，又怕漏掉什么
- 一段内容里同时包含解释、脆弱、价值观、自我照顾等很多线索

流程：

1. 选这段内容的大致形状
2. 手动选择自己收到的线索
3. 看见混合情绪
4. 为每条线索选择处理方式
   - 我先收到了
   - 这条需要回应
   - 放进稍后
   - 不需要回应
5. 选择一个足够的回应方向
   - 只确认我收到了
   - 先回应最重要的一条
   - 先表达感谢 / 被触动
   - 问一个开放问题
   - 反映我听到的重点
   - 先不回复，存下
   - 去草稿自检
   - 先回到自己

可保存内容：

- 1-8 个发现点
- 稍后话题

这个流程不会自动总结消息，也不会分析对方心理。

### 场景 5：我想记录一次互动

入口：首页 -> `记录互动`，或记录页 -> `存下新的记录`

适合这些内容：

- 一次具体互动
- 一段对话后的感受
- 一个想之后再看的事实
- 从触发流程带来的快速记录
- 从草稿自检转来的私下记录

记录字段：

- 短标题
- 可确认的事实
- 我脑中出现的解释
- 情绪
- 身体
- 连接感
- 激活程度
- 可观察的连接证据，或和自己的真实接触证据
- 下一步
- 对能量的影响

保存前会预览这次可能进入哪些储蓄罐。保存后可以在记录详情和账户明细里查看来源和原因。

草稿规则：

- 有实质内容时会自动保存本地草稿
- 返回后再进入可恢复未完成记录
- 草稿只在本机保存

### 场景 6：我需要先回到自己

入口：首页 -> `回到自己`，或从高激活流程进入

适合这些时刻：

- 身体很紧
- 情绪很高
- 不适合继续分析
- 想先做一个不会加重的动作

流程：

1. 选一个身体落点
   - 喝水
   - 洗手 / 洗脸
   - 脚踩地面
   - 慢慢呼气
   - 五感观察

2. 带一句锚点
   - 可以用已有锚点
   - 也可以改成自己的话
   - 可选择保存为锚点

3. 回到一个现实动作
   - 先停止分析
   - 泡一杯喝的
   - 洗澡
   - 手机放远一点
   - 做一件小事

4. 看能量变化
   - 轻一点
   - 差不多
   - 更重
   - 说不清

回到自己只会影响「自己」和「能量」，不会自动创建「连接」影响。

### 场景 7：我想把线索放进稍后

入口：底部导航 -> `稍后`

稍后用于存：

- 发现点
- 想探索的问题
- 稍后话题
- 行动想法

你可以：

- 手动存一个发现点
- 选择类型和主题
- 加一句备注
- 加一个想探寻的问题
- 筛选全部、发现点、想理解、想分享、先放着、已回看、不用了
- 打开详情回看来源、补记、保存锚点、更新状态
- 从详情把一个发现点转成小练习

稍后没有 due date、逾期、欠账、连续打卡或未完成压力。

保存、回看或转成小练习默认不会改变储蓄罐。只有之后在练习详情里记录一次练习，才可能产生透明的「自己」或「能量」影响。

### 场景 8：我想做一个很小的练习

入口：底部导航 -> `练习`

练习用于存一个轻量行动，不是打卡计划。

可以从两种方式创建：

1. 从推荐小动作创建
   - 系统给一个推荐动作和两个备选
   - 可以换一个
   - 可以存成小练习

2. 自己建一个小练习
   - 我想练习什么
   - 小到今天能试一次的动作是什么
   - 什么算练习过一次

3. 从稍后发现点创建
   - 在发现点详情里确认三句预填内容
   - 存成小练习后进入练习详情
   - 创建本身不改变储蓄罐

记录一次练习时，可以选择：

- 完成了
- 完成一部分
- 只是看见了
- 不适合

影响规则：

- 创建练习本身不改变储蓄罐
- 完成、部分完成、只是看见，可以创建透明的「自己」影响
- 完成了也可能创建一点「能量」恢复
- 不适合不会产生负面影响

当前限制：

- 暂未支持暂停、退休、编辑、保存为想法。
- 没有 streak、补卡、惩罚或连续天数压力。

## P2 轻量觉察分支

这些分支是短工具，不是课程，也不是诊断。大多数分支完成后会保存一个发现点到「稍后」，默认不改变储蓄罐。

### 校准一个情绪

入口：首页 -> `校准一个情绪`

用途：

- 把恐惧、羞耻、愤怒、脆弱等情绪当作信使
- 看它可能在保护什么
- 看它可能推你做什么过快反应
- 选一个更有智慧的小动作

不会要求你消灭情绪，也不会强行正向解释。

### 连接恒常

入口：信号检查、草稿自检、情绪校准、健康关系学习等相关路径

用途：

- 区分「连接感塌了」和「事实证明连接消失」
- 保存曾经存在的可观察连接证据
- 给自己一个延迟、回到自己或稍后再看的动作

不会诊断依恋类型，也不会生成客体恒常性分数。

### 边界清晰

入口：草稿自检、健康关系学习、自我关怀、主体性转换等相关路径

用途：

- 区分什么是我的，什么不是我的
- 选择一个边界形式
- 留下一个由自己完成的下一步

不会提供法律建议、危险评分，也不会把边界变成控制、惩罚或测试。

### 自我关怀

入口：回到自己、草稿自检、旧感觉、小练习结果等相关路径

用途：

- 看见哪里很难
- 识别内部审判者
- 把严厉自我攻击改成诚实但不攻击的话
- 选择一个温和但负责的动作

不会创建自尊、自我价值、吸引力或讨喜程度分数。

### 旧感觉 / 内部审判者

入口：草稿自检、情绪校准、自我关怀等相关路径

用途：

- 区分今天的「蚊子」和被碰到的「大象」
- 看见被触到的需要
- 看见旧保护程序想让你做什么
- 选择一个现在的自己能做的回应

不会要求你回忆创伤细节，也不会判断童年来源。

### 主体性转换

入口：草稿自检、边界相关路径

用途：

- 从受害者、拯救者、迫害者姿态里拿回一点主体性
- 转向创造者、引导者、挑战者
- 选择一个自己能完成的动作

这里只标记自己的姿态，不给对方贴标签。

### 健康关系学习

入口：复杂来信、草稿自检、边界清晰、情绪校准等相关路径

用途：

- 看这一刻更像吸引、幻灭、自省、修复、磨合、启示或普通照顾
- 区分照顾、依恋焦虑、控制、新鲜感、真实修复
- 选择修复、边界、等待、稍后或回到自己

不会做真爱测试、兼容性评分、去留建议或关系判决。

### 看见 / 被看见

入口：复杂来信、草稿自检等相关路径

用途：

- 记录自己被看见的可观察证据
- 练习更准确地看见对方
- 选择一个微动作：反映重点、问开放问题、先不建议、问我漏掉了什么
- 检查自己是否还有容量继续听或回应

不会分析对方人格，也不会要求你一直接住对方。

## 设置和数据管理

入口：底部导航 -> `设置`

可以做：

- 编辑当前空间名称和描述
- 查看本地保存说明
- 查看存储状态和最近保存时间
- 查看导出 / 导入占位
- 删除本地数据

删除本地数据后：

- 当前浏览器里的空间、记录、发现点、练习、锚点都会清空
- 应用会回到首次设置

## 项目结构

```text
src/
  App.tsx                    路由和应用壳
  components/                通用 UI 组件
  copy/                      复用文案
  domain/                    纯业务规则、派生逻辑和测试
  routes/                    页面级流程
  storage/storageAdapter.ts  localStorage 边界
  store/AppStoreContext.tsx  应用写入边界
  styles/                    全局样式和 token
```

重要约束：

- 不要在 `storageAdapter.ts` 之外直接使用 `localStorage`。
- 持久化写入必须通过 `AppStoreContext` actions。
- 账户摘要从保存的影响事件派生，不单独保存权威余额。
- 新增心理相关功能必须保持轻量、可跳过、非诊断。
- 用户文案优先使用「存下」「储蓄罐」「小动作」「练习」「发现点」「稍后」这类低压语言。

## PWA 行为

当前提供 manifest、移动端 PWA 外壳和基础 service worker。

首次成功加载后，浏览器会缓存应用外壳和静态资源，让应用可以在离线时重新打开。用户数据仍只保存在当前浏览器的 `localStorage` 中，不做同步或备份。
